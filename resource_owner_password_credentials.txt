Не рекомендован для использования.
Используется для авторизации доверенных сторонних приложений или кастомных oauth решений.
Для получения access токена используются только логин и пароль пользователя
Grant Type: password
Flow:
I. Клиент отправляет POST запрос:
    1. grant_type=password
    2. username=
    3. password=
    4. scope=

II. Если данные прошли проверку на сервере авторизации,
сервер авторизации отправляет клиенту access_token, token_type, expires_in, refresh_token.
